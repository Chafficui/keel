# Keel — Brand Guidelines

## 1. Brand Essence

**Keel** is a developer tool brand centered on **structure, clarity, and reliability beneath complexity**.

### Core Idea
Like a ship’s keel, Keel is the invisible foundation that keeps systems stable, aligned, and moving forward.

### Brand Attributes

- **Precise** — sharp, intentional, engineered
- **Minimal** — no unnecessary abstraction
- **Reliable** — stable under pressure
- **Modern** — built for current dev workflows
- **Quietly powerful** — avoids noise, delivers impact

---

## 2. Visual Identity

### Logo

- Abstract geometric keel shape
- Sharp angles, no curves
- Flat design, no effects
- No text ever paired with the mark

### Logo Usage

- Use standalone as icon (CLI, npm, UI)
- Maintain clear space (0.5× height)
- Prefer centered or slightly bottom-weighted placement

### Variants

- Primary (dark navy + electric blue accent)
- Monochrome (black / white)

---

## 3. Color System

### Primary Colors

- **Keel Navy**: `#0B1220`
- **Keel Blue**: `#2F80FF`

### Neutral Scale

- `#FFFFFF` (white)
- `#E5E7EB` (light gray)
- `#9CA3AF` (mid gray)
- `#111827` (dark gray)

### Usage Principles

- Dark-first design
- Accent blue used sparingly (focus, interaction, highlights)
- No gradients, no glow effects in UI (only flat)

---

## 4. Typography

### Style Direction

- Clean, modern sans-serif
- High readability in terminal + UI
- Slightly technical feel

### Recommended Fonts

- UI: Inter / system-ui
- Code: JetBrains Mono / Fira Code

### Rules

- Avoid decorative fonts
- Keep hierarchy simple (weight > size differences)

---

## 5. Layout & Spacing

### Principles

- Generous whitespace
- Grid-based layout
- Clear visual hierarchy

### Spacing System

Use consistent spacing scale (e.g.):

- 4 / 8 / 12 / 16 / 24 / 32 px

### Alignment

- Prefer left alignment in UI
- Centering only for hero or logo-focused sections

---

## 6. Iconography & Graphics

### Style

- Geometric
- Minimal strokes or filled shapes
- Consistent angle language with logo

### Do

- Reuse logo angles in UI elements
- Keep icons simple and readable

### Don’t

- Use skeuomorphic or detailed illustrations
- Mix multiple icon styles

---

## 7. Motion (Optional but Recommended)

### Style

- Fast, subtle, purposeful
- No playful or bouncy motion

### Examples

- Slight slide or fade
- Split/reveal animations (echoing keel shape)

---

## 8. Tone of Voice

### Characteristics

- Direct
- Technical but approachable
- Confident, not hype-driven

### Examples

**Good**
- "Define your structure."
- "Ship with confidence."
- "Less guesswork, more control."

**Avoid**
- Marketing buzzwords
- Overly playful language
- Long explanations when short works

---

---

## 10. Website Guidelines

### Style

- Dark theme by default
- Minimal hero section
- Focus on product, not decoration

### Sections

- Hero (clear value proposition)
- Quick start
- Features (structured, not fluffy)
- Code examples

### Interaction

- Subtle hover states
- No heavy animations

---

## 11. Example Projects

### Philosophy

- Show real usage, not toy examples
- Prioritize clarity over completeness

### Structure

- Minimal setup
- Clean folder structure
- Clear entry point

### Documentation

- Short explanations
- Focus on "why" + "how"

---

## 12. Do / Don’t Summary

### ✅ Do

- Keep everything minimal and structured
- Use strong contrast and clean geometry
- Prioritize usability over decoration

### ❌ Don’t

- Overdesign or overanimate
- Add unnecessary abstractions
- Break consistency between CLI, web, and docs

---

## 13. Brand Consistency Rule

Everything that belongs to Keel should feel like:

> "Engineered, not decorated."

If something feels flashy, noisy, or ornamental — it’s off-brand.
